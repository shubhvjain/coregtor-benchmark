# Parameter Selection of CoRegTor

The goal is to find a default parameter selection values. 

We use various validation indices to measure the performance of the results generated by each configuration. 

### File locations

The files for this experiment reside in the following folders in the project:
- `experiments/ps` : this folder has the experiment files for individual datasets where all commands are added. 
- `results/ps`: this folder contains the result files. Each dataset has a separate folder. 
- `analysis/ps`: this folder contains the analysis of the parameter selection include final figures.

### Parameters

We generate results using multiple parameters. There are 96 cases. There cases are defined in this file: [experiments/ps/ps_configs.csv](/experiments/ps/ps_configs.csv) 

The 2 main parameters of CoRegTor are:
- `distance_measure` :  the distance measure used to compare the gene frequency histogram of pairwise root genes. We consider the following measures:
  - euclidean_distance
  - manhattan_distance
  - canberra_distance
  - chebyshev_distance
  - cosine_distance
  - weighted_jaccard
  - sorensen_distance
  - jensenshannon_distance
- `clustering_method` : the clustering method used to find coregulatory modules from the pair wise distance matrix generated in the previous step. We consider the following clustering methods:
  -  hierarchical clustering
  -  hdbscan
  -  community_detection


### Input Target list

We consider the top expressed 500 protein coded genes as targets genes. CoRegTor is run on each 500 target genes.

### Input Source list 

We consider the top expressed 2250 regulators as source genes for each selected target. 


## Commands to generate all results

Results generation is divided in 2 parts:
- generation of clusters for each dataset, for each config 
- computation of validation indices for each dataset, for each config

### Experiment location 
```bash
export EXP_PATH_AMY="$PWD/experiments/ps/amy.json"
export EXP_PATH_BLA="$PWD/experiments/ps/bla.json"
export EXP_PATH_BLD="$PWD/experiments/ps/bld.json"
export EXP_PATH_CDN="$PWD/experiments/ps/cdn.json"
export EXP_PATH_HRA="$PWD/experiments/ps/hra.json"
export EXP_PATH_KDC="$PWD/experiments/ps/kdc.json"
export EXP_PATH_LIV="$PWD/experiments/ps/liv.json"
export EXP_PATH_LUN="$PWD/experiments/ps/lun.json"
export EXP_PATH_PIT="$PWD/experiments/ps/pit.json"
export EXP_PATH_STM="$PWD/experiments/ps/stm.json"

export NJOBS=32
export ENV_PATH="$PWD/.env"
export EXP_PATH="$PWD/experiments/ps/amy.json"
```

### Run CoRegTor for a dataset

```bash
poetry run coregtor_pipeline bulk batch  --config env=$ENV_PATH  input=$EXP_PATH items=500
```
This will generate results files for each target in the temp folder (subfolder `results`, one `.pkl` file per target). 

### Generate Result for a dataset 

```bash
poetry run result --env=$ENV_PATH  --input=$EXP_PATH --id=r1  --njobs=$NJOBS
poetry run result --env=$ENV_PATH  --input=$EXP_PATH --id=r2  --njobs=$NJOBS
```

The first command generate a summary file `target_stats.json` in the exp results folder in the result directory. this file contains stats about each target ran. The second command generates a subfolder `clusters` in the temp file. It creates a cluster file for each configuration. The configurations are specified in the exp file.  

### Generate Validation indices for a dataset

This process is divided into different command to allow them to run in parallel.

```bash
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=index1  --njobs=$NJOBS
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=dcorrindex  --njobs=$NJOBS
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=index_tfbs  --njobs=$NJOBS
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=ppiindex1  --njobs=$NJOBS
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=ppiindex2  --njobs=$NJOBS
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=rfreq  --njobs=$NJOBS
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=freqindex  --njobs=$NJOBS
```
Finally, generate combined index file:

```bash
poetry run result  --env=$ENV_PATH  --input=$EXP_PATH --id=r4  --njobs=$NJOBS
```
